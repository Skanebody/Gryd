# GRYD — Motion Design, univers visuel, badges, médailles, jauges et objets virtuels

## Objectif du document

Ce document définit l’univers visuel vivant de **GRYD** :
- animations ;
- motion design ;
- feedback dans l’app ;
- résultats animés ;
- chiffres vivants ;
- victoires ;
- badges ;
- jauges d’avancement ;
- médailles de classement ;
- emojis / emotes ;
- objets virtuels in-app ;
- système de rareté ;
- cohérence visuelle globale autour de la course.

L’objectif n’est pas de faire une app statique.

L’objectif est de créer :

```txt
un monde vivant
+ premium
+ sportif
+ gaming
+ lisible
+ cohérent
+ désirable
```

Inspiration souhaitée :
- le niveau d’exécution d’une marque comme **Nike**, **On Running**, **Adidas Running** ;
- la clarté et le plaisir d’usage des jeux mobiles premium ;
- une direction plus moderne, plus noire, plus nocturne, plus “performance urbaine”.

GRYD ne doit pas ressembler à une app fitness générique.

GRYD doit ressembler à :

```txt
une marque de course premium
+ un jeu de territoire vivant
+ une interface mobile énergique
```

Nom public : **GRYD**  
Baseline : **Cours. Capture. Défends.**  
Positionnement : **Le jeu de conquête de territoire pour run clubs.**

# 1. Vision créative

## 1.1 Intention générale

GRYD doit donner la sensation de :
- vitesse ;
- tension ;
- respiration ;
- conquête ;
- précision ;
- progression ;
- récompense ;
- statut.

Le monde GRYD doit être visuellement cohérent autour de l’univers de la course.

Le motif central n’est pas l’épée, le tank ou l’armure.

Le motif central est :

```txt
la foulée
la ligne
le souffle
l’impact au sol
la trajectoire

la performance

```

# 2. Big idea : le système **Shoe Rank**

## 2.1 Concept

Plutôt qu’un système de rareté générique, GRYD peut s’appuyer sur un système de progression “chaussure”.

Le joueur ne débloque pas juste des badges.

Il gravit des niveaux d’identité running.

Exemple :

| Niveau | Nom | Univers visuel |
|---|---|---|
| 1 | **Bronze Runner** | base, discipline, départ |
| 2 | **Silver Runner** | régularité, maîtrise |
| 3 | **Gold Runner** | conquête, confiance |
| 4 | **Carbon Runner** | performance, vitesse |
| 5 | **Elite Runner** | prestige, domination |
| 6 | **Legend Runner** | statut absolu |

Je recommande d’éviter “diamant” ou “mythique” trop gaming fantasy.  
Le running premium appelle un vocabulaire plus produit/performance.

---

## 2.2 Option recommandée

Au lieu de :
- Bronze ;
- Silver ;
- Gold ;
- Platinum ;
- Diamond ;

je recommande :

| Palier | Nom | Référence running |
|---|---|---|
| 1 | **Road** | premiers kilomètres |
| 2 | **Tempo** | régularité |
| 3 | **Race** | vitesse |
| 4 | **Carbon** | performance premium |
| 5 | **Elite** | haut niveau |
| 6 | **Legend** | statut iconique |

Mais pour la lisibilité grand public, on peut afficher les deux :

```txt
Carbon — Gold Tier
Elite — Platinum Tier
```

### Recommandation finale

Pour le MVP / V1, utiliser :

```txt
Bronze

```

Car :
- simple ;

- premium ;
- compatible avec une direction running.

# 4. Principes de motion design

## 4.1 Motion philosophy

Le motion design GRYD doit être :

```txt
rapide
lisible
satisfaisant
énergique
précis
jamais gadget
```

Il doit évoquer :
- la foulée ;
- l’élan ;
- la respiration ;
- l’impact au sol ;
- la propulsion ;
- la progression.

---

## 4.2 Verbes de motion à utiliser

GRYD bouge avec 6 verbes :

1. **Pulse** → respiration, attention, objectif
2. **Sweep** → capture, balayage, progression
3. **Bounce** → victoire, compteur, reward
4. **Slide** → navigation, panels, progression
5. **Glow** → rareté, récompense, statut
6. **Burst** → capture, badge, victoire

---

## 4.3 Timings recommandés

- micro interaction : **120–220 ms**
- transition standard : **220–320 ms**
- compteur de résultat : **450–900 ms**
- révélation de récompense : **700–1200 ms**
- animation de victoire : **900–1600 ms**
- animation rare / premium : **1200–2200 ms**

Courbes :
- ease-out propre ;
- spring douce ;
- ressort rapide sur les chiffres ;
- inertie légère sur les cartes et bottom sheets.

# 5. Où mettre des animations

GRYD doit bouger à plusieurs endroits.

## 5.1 Sur la carte

Animations à prévoir :
- pulsation de la position du joueur ;
- glow doux de la zone crew ;
- apparition d’un hex capturé ;
- vague de capture quand plusieurs hexes sont pris ;
- halo sur objectif crew ;
- pulse sur zone contestée ;
- decay visuel progressif ;
- lock visuel discret ;
- bouclier avec animation de membrane.

## 5.2 Pendant une course live

Animations à prévoir :
- compteur km qui s’incrémente proprement ;
- compteur hexes capturés qui pulse ;
- feedback haptique + flash léger sur capture ;
- mini burst autour du bouton pause ;
- ligne de trace qui se dessine en temps réel ;
- micro pulse “course propre” / “GPS faible”.

## 5.3 Résultat de course

C’est la zone la plus importante.

Animations à prévoir :
- reveal du score ;
- compteur des points ;
- compteur des hexes ;
- apparition des badges ;
- animation de performance bonus ;
- montée de jauges ;
- déverrouillage de récompenses ;
- confetti / particle FX premium pour victoire forte.

## 5.4 Crew

Animations à prévoir :
- progression offensive ;
- compteur collectif ;
- carte de zone qui se remplit ;
- réactions / emotes chat ;
- badge “crew en tête” ;
- ouverture d’un coffre crew.

## 5.5 Boutique

Animations à prévoir :
- rotation légère des objets ;
- glow de rareté ;
- reveal de skin ;
- ouverture d’un pack ;
- comparaison avant/après ;
- prévisualisation d’un objet sur carte / profil.

## 5.6 Profil / Performance

Animations à prévoir :
- jauges de progression ;
- Score Forme qui se remplit ;
- records battus ;
- timeline de badges ;
- medals reveal ;
- graphiques avec animation d’entrée.

# 6. Écrans clés à animer

## 6.1 Résultat de course — structure idéale

Ordre d’apparition recommandé :

### Étape 1 — Reveal principal

```txt
Course validée
```

ou

```txt
Territoire conquis
```

Animation :
- fondu + slide up ;
- son doux / haptique.

### Étape 2 — Chiffre héros

Exemple :

```txt
+214 HEXES
```

Animation :
- chiffre en gros ;
- bounce contrôlé ;
- glow léger ;
- petites particules de vitesse.

### Étape 3 — Détails

Exemples :
- +62 hexes volés
- +34 hexes défendus
- +1240 pts
- +56 Foulées
- +7 % performance

Animation :
- compteurs qui roulent ;
- apparition en cascade ;
- petites icônes animées.

### Étape 4 — Progression

Afficher :
- jauge niveau joueur ;
- progression Pass ;
- progression badge ;
- progression crew.

Animation :
- remplissage fluide ;
- seuils déclenchent un burst.

### Étape 5 — Reward moment

Si record, badge ou rang débloqué :
- carte reward premium ;
- glow ;
- médaille ;
- son plus valorisant.

### Étape 6 — CTA

```txt
Partager
Retour carte
Voir performance
```

---

## 6.2 Victoire / domination

Si un joueur ou crew gagne une zone stratégique, créer une animation plus forte.

Exemple :
- vignette secteur ;
- sweep chartreuse sur la carte ;
- titre :

```txt
République contrôlée
```

- médaille ou badge zone ;
- montée d’un compteur de contrôle ;
- mini explosion d’hexes maîtrisée.

---

## 6.3 Déblocage de badge

Animation :
- badge au centre ;
- rotation légère ou pop ;
- halo de rareté ;
- nom du badge ;
- phrase d’impact.

Exemple :

```txt
Nouveau badge
LONG RUN
10 km validés
```

# 7. Jauges à créer

GRYD a besoin de jauges partout.

## 7.1 Types de jauges

### Jauge de niveau

Pour le joueur :
- XP ;
- niveau ;
- rang.

### Jauge de Pass

Pour la saison :
- progression de paliers ;
- reward à venir ;
- bonus premium.

### Jauge d’offensive

Pour un crew :
- objectif total ;
- progression équipe ;
- temps restant.

### Jauge de contrôle secteur

Pourcentage de domination d’un secteur :
- 12 % présence ;
- 38 % contesté ;
- 62 % contrôlé.

### Jauge de streak

Régularité hebdo / mensuelle.

### Jauge de Score Forme

Page performance.

### Jauge de fiabilité

Données fiables / partiellement validées.

---

## 7.2 Style des jauges

Le style des jauges doit évoquer :
- semelle ;
- piste ;
- capsule sport ;
- énergie.

Exemples de styles :
- barre capsule arrondie ;
- barre “rail” avec segmentations ;
- anneau progressif autour d’un chiffre ;
- jauge semelle de chaussure stylisée ;
- jauge trajectoire incurvée.

### Recommandation forte

Créer une jauge signature GRYD inspirée de la **semelle de running** :
- forme longue, légèrement arquée ;
- segments qui s’allument ;
- texture discrète type outsole ;
- glow rareté en fin de remplissage.

C’est très marque, très mémorable, et totalement cohérent.

# 21. Erreurs à éviter

1. Faire des animations gadget.
2. Faire trop gaming fantasy.
3. Faire cheap ou trop “template”.
4. Trop d’effets partout.
5. Manque de cohérence entre badges, shop et carte.
6. Objets in-app sans rapport avec le running.
7. Médailles trop génériques.
8. Rareté pas lisible.
9. Résultats de course trop fades.
10. Pas assez de récompense visuelle.
11. UI trop figée.
12. Copier Strava.
13. Copier Clash of Clans visuellement.
14. Utiliser des emojis trop enfantins.
15. Oublier la sensation “marque premium”.

# 22. Résumé final

GRYD doit créer un vrai monde visuel.

Pas juste :
- une carte ;
- des chiffres ;
- une boutique.

Mais :

```txt
un univers premium de conquête par la course
```

La règle de cohérence :

```txt
Si Nike Running ou On Running avaient créé un jeu de territoire mobile,
il pourrait ressembler à GRYD.
```

La signature créative recommandée :

```txt
Chaussure de performance
+ trajectoire
+ semelle
+ vitesse
+ rareté premium
+ reward satisfying
```

Phrase finale :

```txt
Chaque course doit laisser une trace.
Chaque victoire doit se ressentir.
Chaque récompense doit se désirer.
```
